## For Hardik (transactional-spine tables)
- **Every table migration must GRANT explicitly** (auto-expose is OFF):
  ```sql
  grant all privileges on table public.<t> to service_role;       -- our server
  grant select on table public.<t> to authenticated;              -- + insert/update where the role legitimately writes
  ```
  Without it you get **"permission denied"** even with the service-role key. RLS
  applies on top (auto-RLS enables it — add policies).
- **Apply migrations** via the Supabase **SQL Editor** (files in `supabase/migrations/`).
- **Scripts under tsx:** no top-level `await` (wrap in `main()`); `@/*` aliases may
  not resolve — use relative imports; load `.env.local` manually; never print keys.
- Reuse `src/lib/supabase/admin.ts` for server reads/writes. The user-session
  client + session middleware land with the Auth module (M05–M09, shared — let's
  coordinate so we build it once).
